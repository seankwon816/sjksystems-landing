01 / Thesis

What we're building.

Agents are already making decisions, spending money, and talking to other agents on people's behalf. But anyone running one runs into the same two problems. First, agents waste most of their time and tokens not on hard reasoning but on mundane failures. Losing files, letting context bloat, looping on small mistakes, burning cost. Second, there is still no standard way for one person's agent to find, reach, and talk to another's.

SJK Systems builds one product for each. ClevAgent is the workstation you run your agent inside, with our own AI watching the session and stepping in when it sees a problem. ForAgent is the network your agent uses to reach someone else's. A carrier that routes, authenticates, and logs the call without reading a word of it.

02 / Products

One to work with your agent. One to reach everyone else's.

ClevAgent

v0.x · desktop
Beta

A workstation for running your agent. ClevAgent watches the session for the inefficient patterns every agent hits. Bad file paths, bloated context, memory leaks, loops, runaway spend. When one trips, our AI injects a message into the terminal and puts the agent back on track. Security and audit controls are next.

  • File path & memory hygiene
  • Loop detection & recovery
  • Cost & context ceilings
  • Security & audit (coming)

ForAgent

v0.x · protocol
Private beta

A carrier network for agents. Publish your agent at a public address, approve incoming dials, and talk through a bounded session. We route, authenticate, log, and bill the call, but we never read the contents. Payload bodies null out after 30 days.

  • Public agent addresses
  • Dial, ring, accept, hang up
  • Routing, auth, billing, audit
  • Payload-blind, 30-day retention
